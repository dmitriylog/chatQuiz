# Инструкция: запуск чата для двух пользователей

## Что означает ошибка `Errno 111 Connect call failed`

**Кратко:** программа не смогла подключиться к серверу по указанному адресу и порту — соединение **отклонено**.

На вашем скриншоте клиент пытался открыть:

`158.160.231.28` **порт 8765** (и с галочкой WSS).

На типичной установке в **Yandex Cloud**:

| Порт | Доступен с интернета? |
|------|------------------------|
| **80** | Да (nginx → чат) |
| **8765** | Нет (только внутри сервера) |

Поэтому `8765` снаружи даёт **Errno 111** — на этом порту никто не слушает запросы из интернета.

### Что указать в окне входа (Yandex Cloud)

| Поле | Значение |
|------|----------|
| Адрес сервера | `158.160.231.28` (без `:8765`) |
| Ваше имя | любое, разное у двух людей |
| Пароль | только если задавали `CHAT_PASSWORD` на сервере |
| **WSS** | **Снять**, если заходите по `http://` (без сертификата HTTPS) |

После обновления клиента для облачного IP автоматически используется порт **80**.

## Вариант A — тест на одном компьютере (два терминала + клиенты)

Подходит, чтобы проверить, что всё работает, без облака.

### Терминал 1 — сервер

```bash
cd /home/d1m1d1n/chat
source .venv/bin/activate
pip install -r requirements.txt
python run_server.py --host 0.0.0.0
```

Должно появиться: `Uvicorn running on http://0.0.0.0:8765`  
**Это окно не закрывайте.**

### Терминал 2 — первый пользователь

```bash
cd /home/d1m1d1n/chat
source .venv/bin/activate
python main.py
```

В окне входа:

- Адрес: `127.0.0.1`
- Имя: `Алиса`
- WSS: **выключено**

Нажмите **Войти**.

### Терминал 3 — второй пользователь

Те же команды, что в терминале 2:

```bash
cd /home/d1m1d1n/chat
source .venv/bin/activate
python main.py
```

- Адрес: `127.0.0.1`
- Имя: `Боб`
- WSS: **выключено**

В списке **Участники** оба увидят друг друга и смогут писать в общий чат или открыть личный (клик по имени).

## Вариант B — два компьютера через Yandex Cloud

### Шаг 1. Сервер в облаке (один раз)

На ВМ в Yandex Cloud (по [deploy/DEPLOY_YANDEX_CLOUD.md](deploy/DEPLOY_YANDEX_CLOUD.md)):

- запущен Docker + nginx;
- в группе безопасности открыт **TCP 80**;
- публичный IP, например `158.160.231.28`.

Проверка с вашего ПК в браузере:  
`http://158.160.231.28/` — должна открыться страница (или пустой ответ, но не «нет соединения»).

На сервере при необходимости:

```bash
cd /opt/chat
sudo docker compose ps
sudo docker compose up -d
sudo systemctl status nginx
```

### Шаг 2. Первый компьютер — клиент

```bash
cd /home/d1m1d1n/chat
source .venv/bin/activate
python main.py
```

| Поле | Значение |
|------|----------|
| Адрес | `158.160.231.28` |
| Имя | `DmitriyS` |
| WSS | выкл. (пока нет HTTPS) |

### Шаг 3. Второй компьютер — клиент

Тот же `python main.py`, **тот же IP**, **другое имя** (например `Друг`).

## Вариант C — два компьютера в одной Wi‑Fi (без облака)

### Терминал на ПК, который будет «сервером»

Узнайте локальный IP (Linux):

```bash
hostname -I
```

Например `192.168.1.50`. Запуск:

```bash
python run_server.py --host 0.0.0.0
```

Разрешите порт **8765** в файрволе, если он включён.

### На обоих ПК — клиент

- Адрес: `192.168.1.50` (IP сервера, не `127.0.0.1` на втором ПК)
- Разные имена
- WSS: выкл.

Если не подключается — укажите явно: `192.168.1.50:8765`

## Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Errno 111` … `8765` | Порт 8765 закрыт снаружи (облако) | IP без `:8765`, WSS выкл., порт 80 |
| `Errno 111` … `80` | Сервер/nginx не запущен | `docker compose up`, `nginx`, группа безопасности |
| SSL / certificate error | Включён WSS без HTTPS | Снять галочку WSS |
| Неверный пароль | На сервере задан `CHAT_PASSWORD` | Ввести тот же пароль или убрать переменную |

## Краткая схема

```
Терминал 1 (сервер):  python run_server.py --host 0.0.0.0
                              ▲
Терминал 2 (клиент):  python main.py  → 127.0.0.1, имя 1
Терминал 3 (клиент):  python main.py  → 127.0.0.1, имя 2
```

Облако: сервер на ВМ, оба клиента → `158.160.231.28`, порт 80, WSS выкл.
