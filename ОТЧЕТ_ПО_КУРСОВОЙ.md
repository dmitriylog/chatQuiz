Введение

Актуальность темы: в современном мире мессенджеры стали неотъемлемой частью повседневного общения. Однако пользователям недостаточно простого обмена текстовыми сообщениями — они хотят интерактивного взаимодействия, включая развлекательные элементы. Разработка мессенджера со встроенной викториной объединяет в себе задачи сетевого программирования, безопасности данных, облачных технологий и создания удобного пользовательского интерфейса.

Ключевые аспекты проекта:

Сетевое взаимодействие: организация стабильной связи между клиентами через WebSocket в режиме реального времени.

Безопасность: защита учётных записей через хэширование паролей и шифрование сообщений.

Облачные технологии: развёртывание сервера на Яндекс.Облаке для круглосуточной доступности.

Удобство для пользователя: создание интуитивного графического интерфейса и автоматического установщика.

Цель курсовой работы — проектирование, разработка и развёртывание в облачной инфраструктуре отказоустойчивого сетевого приложения «Мессенджер с интегрированной викториной», обеспечивающего безопасную авторизацию пользователей, обмен сообщениями и игровой процесс в режиме реального времени.

Задачи:

Спроектировать архитектуру базы данных для хранения учётных записей, сообщений и игровой статистики в СУБД SQLite.

Реализовать серверную часть на Python с использованием FastAPI и WebSocket для обработки подключений в реальном времени.

Разработать клиентское приложение с графическим интерфейсом на базе customtkinter, обеспечивающее визуализацию чата и викторины.

Интегрировать механизмы информационной безопасности: хэширование паролей (PBKDF2), шифрование сообщений (Fernet).

Развернуть серверную часть на виртуальной машине в Яндекс.Облаке, настроить сетевые экраны и удалённое администрирование.

Автоматизировать сборку клиентской части с формированием исполняемого инсталлятора (.exe) для конечных пользователей.

2) Технологический стек

2.1. Язык программирования: Python

Выбор Python обоснован:

Кроссплатформенность: код работает на Linux (сервер) и Windows (клиент).

Богатые библиотеки: FastAPI, customtkinter, cryptography, sqlite3.

Быстрая разработка: динамическая типизация и читаемый синтаксис.

2.2. Серверный фреймворк: FastAPI

FastAPI выбран за:

Асинхронную архитектуру: обработка тысяч подключений через WebSocket.

Автоматическую документацию: Swagger UI для тестирования API.

Высокую производительность: сравним с Node.js и Go.

2.3. База данных: SQLite

SQLite используется для:

Простоты развёртывания: не требует отдельного сервера.

Надёжности: транзакции ACID, целостность данных.

Лёгкого резервного копирования: один файл базы данных.

2.4. Графический интерфейс: customtkinter

customtkinter обеспечивает:

Современный дизайн: адаптированные виджеты в стиле Windows 11/macOS.

Тёмную тему: комфорт для глаз при длительной работе.

Адаптивность: автоматическая подстройка под размер окна.

2.5. Безопасность: cryptography

Библиотека cryptography предоставляет:

Fernet (симметричное шифрование): для защиты сообщений.

PBKDF2: для хэширования паролей с солью.

2.6. Облачная инфраструктура: Яндекс.Облако

Виртуальная машина Ubuntu в Яндекс.Облаке обеспечивает:

Доступность 24/7: сервер работает постоянно.

Публичный IP: клиенты подключаются из любой точки мира.

Масштабируемость: возможность увеличения ресурсов при росте нагрузки.

3) Спецификация и назначение классов программного комплекса

3.1. Модуль базы данных (db/)

database.py — основной модуль работы с SQLite:

Инициализация схемы базы данных (таблицы users, messages, quiz_questions, quiz_sessions, friends).

Сохранение и получение сообщений (с шифрованием/дешифрованием).

Управление пользователями: регистрация, аутентификация, профили.

Система друзей: отправка заявок, принятие, отклонение.

Викторина: вопросы, сессии, ответы, статистика.

auth.py — хэширование и проверка паролей:

hash_password(password): хэширование через PBKDF2 с солью.

verify_password(password, hash): проверка соответствия.

encryption.py — шифрование сообщений:

encrypt_message(text): шифрование через Fernet.

decrypt_message(cipher): расшифровка.

3.2. Серверный модуль (server.py)

server.py — основной WebSocket сервер:

Класс Client: представление подключённого пользователя (websocket, user_id, username).

Класс ChatRoom: управление всеми подключёнными клиентами, рассылка сообщений.

Обработчики сообщений:

hello: первичное подключение и аутентификация.

message: отправка сообщения в общий чат.

dm: личное сообщение между пользователями.

quiz_start: запуск викторины.

quiz_answer: ответ на вопрос викторины.

friend_request: отправка заявки в друзья.

accept_friend: принятие заявки.

profile_update: обновление профиля пользователя.

avatar_upload: загрузка аватарки.

quiz_invite: приглашение друга в викторину.

quiz_manager.py — управление викторинами:

Класс QuizManager: логика игрового процесса.

Лобби: ожидание игроков, отсчёт до старта.

Вопросы: случайная выборка из базы, таймер 20 секунд.

Подсчёт очков: правильные ответы, определение победителя.

3.3. Клиентский модуль (client/)

app.py — главное приложение:

Класс ChatApp: основное окно с чатом, списком пользователей, друзьями.

Аутентификация: вход/регистрация.

Чат: общий и личные сообщения.

Друзья: список, заявки, удаление.

Профиль: просмотр и редактирование.

Викторина: запуск, участие, просмотр результатов.

theme.py — цветовая схема:

 COLORS: палитра (фон, акцент, текст, статусы).

Тёмная тема с приглушёнными цветами для комфортного чтения.

profile_dialog.py — диалог профиля:

Отображение информации о пользователе.

Загрузка и удаление аватарки.

Редактирование имени и био.

quiz_window.py — окно викторины:

Отображение вопросов с вариантами ответов.

Таймер обратного отсчёта (20 секунд).

Чат внутри викторины для общения игроков.

Результаты раундов и финальная таблица.

connection.py — сетевое соединение:

Класс ChatConnection: управление WebSocket подключением.

Автоматическое переподключение при обрыве.

Отправка и получение JSON-сообщений.

4) Архитектура системы и сетевое взаимодействие

4.1. Жизненный цикл системы

Шаг 1. Запуск сервера

Сервер запускается на виртуальной машине Яндекс.Облака:

python run_server.py --host 0.0.0.0 --port 8765

Инициализируется база данных SQLite, создаются таблицы.

Сервер слушает WebSocket подключения на порту 8765.

Шаг 2. Подключение клиента

Клиент запускает приложение и вводит адрес сервера (158.160.231.28), логин и пароль.

Устанавливается WebSocket соединение:

ws = websocket.WebSocketApp(f"ws://{server_ip}:8765/ws")

Отправляется сообщение hello с учётными данными.

Шаг 3. Аутентификация

Сервер получает hello, проверяет логин/пароль через DatabaseManager.

При успехе отправляет welcome с данными профиля и списком онлайн-пользователей.

Клиент переходит в главное окно чата.

Шаг 4. Обмен сообщениями

Пользователь отправляет сообщение → клиент посылает {"type": "message", "text": "..."}.

Сервер сохраняет сообщение в базу (с шифрованием) и рассылает всем подключённым.

Получатели видят сообщение в общем чате.

4.2. Сетевой протокол викторины

Инициирование: игрок нажимает "Викторина" → {"type": "quiz_start"}.

Лобби: сервер открывает лобби, ждёт игроков (минимум 2).

Старт: когда игроки готовы → сервер выбирает 10 случайных вопросов из базы.

Вопрос: каждому игроку отправляется {"type": "quiz_question", "question": {...}}.

Ответ: игрок выбирает вариант → {"type": "quiz_answer", "answerIndex": 2}.

Таймер: 20 секунд на вопрос, затем показ правильного ответа.

Финал: после всех вопросов → таблица результатов, определение победителя.

4.3. Система друзей

Отправка заявки: {"type": "friend_request", "to_username": "ivan"}.

Принятие: {"type": "accept_friend", "from_user_id": "abc123"}.

Список друзей: сервер возвращает {"type": "friends_list", "friends": [...]}.

Уведомления: при новых заявках → {"type": "pending_requests_updated"}.

5) Структура базы данных

5.1. Таблица users

Хранит учётные записи пользователей.

Поле	Тип	Описание
id	INTEGER	Первичный ключ, автоинкремент
username	TEXT	Уникальный логин (UNIQUE, NOT NULL)
password_hash	TEXT	Хэш пароля (NOT NULL)
display_name	TEXT	Отображаемое имя
bio	TEXT	Информация о себе
quiz_wins	INTEGER	Количество побед в викторинах
quiz_games	INTEGER	Количество сыгранных игр
quiz_points	INTEGER	Общее количество баллов
created_at	TEXT	Дата регистрации
avatar_data	BLOB	Аватарка (до 1MB)

5.2. Таблица messages

История сообщений общего чата.

Поле	Тип	Описание
id	INTEGER	Первичный ключ
channel	TEXT	Идентификатор канала ("general" или "dm:...")
sender_user_id	TEXT	ID отправителя
sender_username	TEXT	Имя отправителя
body	TEXT	Текст сообщения (зашифрован)
created_at	TEXT	Время отправки

5.3. Таблица quiz_questions

Вопросы для викторины.

Поле	Тип	Описание
id	INTEGER	Первичный ключ
question	TEXT	Текст вопроса
option_a, option_b, option_c, option_d	TEXT	Варианты ответов
correct_index	INTEGER	Индекс правильного ответа (0-3)
sort_order	INTEGER	Порядок отображения

5.4. Таблица quiz_sessions

Сессии викторин.

Поле	Тип	Описание
id	INTEGER	Первичный ключ
status	TEXT	"active" или "finished"
current_index	INTEGER	Текущий вопрос
started_at	TEXT	Время начала
finished_at	TEXT	Время окончания

5.5. Таблица quiz_players

Участники сессии и их очки.

Поле	Тип	Описание
session_id	INTEGER	ID сессии
user_id	TEXT	ID игрока
username	TEXT	Имя игрока
score	INTEGER	Количество правильных ответов

5.6. Таблица friends

Система друзей.

Поле	Тип	Описание
id	INTEGER	Первичный ключ
user_id	TEXT	Отправитель заявки
friend_id	TEXT	Получатель заявки
status	TEXT	"pending" или "accepted"
created_at	TEXT	Дата создания
