# Развёртывание чата в Yandex Cloud

Пошаговая инструкция: создать виртуальную машину (ВМ), открыть порты, залить приложение и открыть чат в браузере по публичному IP.

## Что получится

```
Браузер  →  http://ПУБЛИЧНЫЙ_IP/     (nginx, порт 80)
                ↓
           Docker: uvicorn :8765
                ↓
           WebSocket /ws  — тот же адрес, без отдельного порта
```

Клиент уже подключается к `ws://тот-же-хост/ws` — менять код не нужно.

---

## Шаг 1. Подготовка в консоли Yandex Cloud

1. Войдите в [console.cloud.yandex.ru](https://console.cloud.yandex.ru).
2. Создайте **каталог** (folder), если его ещё нет.
3. Привяжите **платёжный аккаунт** (без него ВМ не запустится; есть пробный период/гранты — смотрите актуальные условия в биллинге).

### SSH-ключ (на вашем ПК)

```bash
# если ключа ещё нет
ssh-keygen -t ed25519 -C "yandex-chat" -f ~/.ssh/yandex_chat -N ""

# посмотреть публичный ключ — его вставите в консоль при создании ВМ
cat ~/.ssh/yandex_chat.pub
```

---

## Шаг 2. Создать виртуальную машину

**Compute Cloud → Виртуальные машины → Создать ВМ**

| Параметр | Рекомендация |
|----------|----------------|
| Имя | `chat-server` |
| Зона | `ru-central1-a` (любая в ru-central1) |
| Образ | **Ubuntu 22.04 LTS** |
| Платформа | Intel / 2 vCPU |
| RAM | **2 ГБ** (для чата достаточно 1 ГБ, но 2 удобнее) |
| Диск | 10–15 ГБ SSD |
| Сеть | Подсеть по умолчанию |
| **Публичный IPv4** | **Включить** (обязательно) |
| Логин | `ubuntu` |
| SSH-ключ | Вставьте содержимое `~/.ssh/yandex_chat.pub` |

Создайте ВМ и дождитесь статуса **Running**. Скопируйте **публичный IP** (например `158.160.12.34`).

---

## Шаг 3. Группа безопасности (открыть порты)

**VPC → Группы безопасности** (или в карточке ВМ → Сеть → группа безопасности).

Добавьте **входящие** правила:

| Протокол | Порт | Источник | Назначение |
|----------|------|----------|------------|
| TCP | 22 | `0.0.0.0/0` или ваш IP `/32` | SSH (лучше только ваш IP) |
| TCP | 80 | `0.0.0.0/0` | HTTP — чат в браузере |
| TCP | 443 | `0.0.0.0/0` | HTTPS (позже, с сертификатом) |

Привяжите группу к сетевому интерфейсу ВМ, если она ещё не привязана.

Проверка с ПК:

```bash
ssh -i ~/.ssh/yandex_chat ubuntu@158.160.12.34
```

---

## Шаг 4. Залить приложение на сервер

На **вашем компьютере** (где лежит проект `chat/`):

```bash
cd /home/d1m1d1n/chat

export CHAT_SERVER=ubuntu@158.160.12.34   # ваш IP
chmod +x deploy/deploy-from-pc.sh deploy/setup-vm.sh
bash deploy/deploy-from-pc.sh
```

Скрипт:

- копирует проект в `/opt/chat` на ВМ;
- ставит Docker, nginx;
- запускает `docker compose up -d`;
- настраивает nginx с поддержкой WebSocket.

### Вручную (если без скрипта)

На сервере после `ssh`:

```bash
sudo mkdir -p /opt/chat
sudo chown -R $USER:$USER /opt/chat
```

С ПК:

```bash
rsync -avz --exclude .venv --exclude .git /home/d1m1d1n/chat/ ubuntu@IP:/opt/chat/
ssh ubuntu@IP 'sudo bash /opt/chat/deploy/setup-vm.sh'
```

---

## Шаг 5. Проверка

В браузере: **http://158.160.12.34/** (ваш IP).

- Введите имя → «Войти в чат».
- Откройте вторую вкладку или телефон в той же Wi‑Fi/мобильном интернете — сообщения должны приходить сразу.

На сервере:

```bash
docker compose -f /opt/chat/docker-compose.yml ps
docker compose -f /opt/chat/docker-compose.yml logs -f
sudo systemctl status nginx
```

---

## Шаг 6 (опционально). Домен и HTTPS

1. Купите домен или используйте поддомен.
2. В DNS создайте **A-запись** на публичный IP ВМ.
3. В `deploy/nginx/chat.conf` замените `server_name _;` на `server_name chat.example.com;`.
4. На ВМ:

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d chat.example.com
```

После этого чат откроется по `https://chat.example.com` — WebSocket автоматически станет `wss://`.

## Обновление приложения

После изменений в коде снова с ПК:

```bash
export CHAT_SERVER=ubuntu@ВАШ_IP
bash deploy/deploy-from-pc.sh
```

Или на сервере:

```bash
cd /opt/chat && git pull   # если используете git
sudo docker compose build && sudo docker compose up -d
```

## Частые проблемы

| Симптом | Решение |
|---------|---------|
| Не открывается сайт | Проверьте группу безопасности (порт 80), публичный IP, `sudo systemctl status nginx` |
| Страница есть, сообщения не идут | WebSocket: `sudo nginx -t`, в логах `docker compose logs` |
| `Permission denied (publickey)` | Неверный ключ при создании ВМ; добавьте ключ в метаданные ВМ |
| Мало памяти | Увеличьте RAM ВМ или `swapoff -a` / добавьте swap 1G |

## Стоимость (ориентир)

Минимальная ВМ (2 vCPU, 2 GB, диск ~10 GB) в ru-central1 — порядка нескольких сотен ₽/мес (зависит от тарифа и скидок). Следите за **Биллинг → Детализация** в консоли.

---

## Безопасность (минимум для интернета)

- SSH только по ключу, порт 22 ограничьте своим IP в группе безопасности.
- Позже добавьте HTTPS (certbot).
- Для продакшена — авторизация, лимиты запросов, не храните секреты в репозитории.

Подробнее про развитие чата — в корневом [README.md](../README.md).
